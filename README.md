# 巴菲特策略美股定投系统

按巴菲特价值投资原则（护城河、长期稳定盈利、低负债、安全边际）从 S&P 500 筛选股票，
并为每月 5000 新币的定投给出具体买入分配建议。

## 安装

```bash
cd /root/market_scan
pip install -r requirements.txt yfinance
```

无需任何 API key（默认 yfinance 数据源免费且无配额）。

## 每月定投流程（核心用法）

每月发工资后跑这三步，全程约 5 分钟（首次扫描约 20-40 分钟）：

```bash
# ① 刷新数据和评分（25 天内扫过的股票自动跳过，增量很快；中断了重跑即续扫）
python main.py scan

# ② 生成本月报告：终端直接给出买什么、买多少，同时生成 HTML 详细报告
python main.py report
#    → reports/report_2026-07.html，浏览器打开可看每只股票的完整分析

# ③ 在券商（IBKR/moomoo/Tiger 等）按建议下单后，把实际成交记录回系统
python main.py buy AAPL 5 --price 230.50 --fx 1.35 --note "7月定投"
#    --price 成交价(USD)  --fx 当时汇率(1 USD 兑多少 SGD，不填自动取当日汇率)
```

`report` 的输出长这样：

```
本月买入建议（预算 5000 SGD ≈ 3700 USD，汇率 1 USD = 1.3514 SGD）：
  GOODCO 现价 $55.00  整股 67 股（$3685.00）或碎股 $3700.00 ≈ S$5000
         [综合分 93.0，安全边际 38.4%]
```

- **整股方案**：适合不支持碎股的券商，余款留到下月
- **碎股方案**：IBKR/moomoo 支持按金额下单，能精确用完预算
- 若显示 **"持币观望"**：说明本月没有股票既是好生意又够便宜——这是策略在正常工作
  （"没有好球就不挥棒"），现金留到下月继续等

## 命令速查

| 命令 | 作用 |
|---|---|
| `python main.py scan` | 扫描 S&P 500 全部成分股并评分 |
| `python main.py scan --tickers AAPL,KO` | 只扫指定股票 |
| `python main.py scan --force` | 忽略 25 天缓存，强制重扫 |
| `python main.py report` | 生成本月 HTML 报告 + 终端买入建议 |
| `python main.py buy AAPL 5 --price 230.5` | 记录一笔实际买入 |
| `python main.py portfolio` | 查看持仓：成本（USD/SGD）、盈亏、占比 |
| `python main.py trades` | 查看全部交易流水 |
| `python main.py delete-trade 3` | 删除录错的第 3 笔交易 |

## HTML 报告怎么读

报告分四部分：

1. **本月买入建议** —— 直接照做的部分：股票、股数、金额（SGD+USD）、当日汇率
2. **候选股排行榜** —— 全部已扫股票按综合分排序；"可买入"= 质量+价格双达标，
   "太贵"= 好公司但当前价格没有安全边际（值得放进观察清单等回调），
   "质量未达标"= 不符合巴菲特标准
3. **个股明细** —— 每只股票 8 项巴菲特标准的实际值和通过/未通过，让你知道"为什么"
4. **持仓快照** —— 你的组合成本、市值、盈亏、集中度

## 策略如何工作

**第一层 · 质量筛选（好生意）**：用近 5 年年报（数据源可得的最大深度），要求
ROE 平均 ≥15% 且每年 ≥10%、毛利率 ≥40%、净利率 ≥10%、EPS 无亏损年且 CAGR ≥5%、
债务/EBITDA <3 或 D/E <0.8、利息覆盖 >5 倍、FCF/净利润 ≥80%；回购加分。

**第二层 · 估值（好价格）**：股东盈余（净利润+折旧摊销−维持性资本开支）两阶段 DCF，
增长率取历史值且上限 10%，折现率 9%，终值增长 2.5%。现价低于内在价值 **25% 以上**
才进入可买入名单，另用盈利收益率对比 10 年期美债做校验。

**第三层 · 综合评分**：质量 70% + 估值 30% 合成 0-100 分。

**每月分配**：5000 SGD 按当日汇率换算，按综合分加权投给评分最高的 1-3 只可买入股票；
单只持仓超过组合 25% 的跳过；没有股票满足安全边际时明确建议持币观望。

## 常用调整（config.yaml）

| 想要什么 | 改哪里 |
|---|---|
| 调整每月定投金额 | `investment.monthly_budget_sgd` |
| 每月买更多/更少只 | `investment.max_buys_per_month`（默认 3） |
| 更严/更松的买入门槛 | `valuation.margin_of_safety_min`（默认 25，调到 40 更保守） |
| 放宽质量标准（候选太少时） | `quality.*` 各阈值，比如 `gross_margin_min: 40` 降到 30 |
| 更保守的估值 | `valuation.discount_rate` 调到 0.10-0.12 |
| 允许单只更集中 | `investment.max_position_pct`（默认 25） |

改完直接重跑 `python main.py scan --force && python main.py report` 即可生效。

## 数据源说明

默认用 **yfinance**（Yahoo Finance）：免费、覆盖全部美股、无每日配额，年报深度约 5 年。
汇率来自欧洲央行（Frankfurter API），S&P 500 成分股列表来自 GitHub 开源数据集。

- 财报/公司概况缓存 90 天，报价/汇率缓存 12 小时，都存在 `data/market_scan.db`
- 已扫描的股票 25 天内不重复扫描（`--force` 可强制），中断后重跑自动续扫

也支持 **FMP**（`config.yaml` 中 `data_source: fmp`）。注意 2025-08 之后注册的 FMP key
免费档只开放少数热门股票的财报、限 250 次/天且历史同样只有 5 年——仅在升级付费档
（10 年财报历史、全股票覆盖）后才值得切换，届时把 `fmp.plan` 改为 `starter`。

## 常见问题

- **扫描中断了怎么办？** 直接重跑 `python main.py scan`，已完成的自动跳过。
- **换了电脑/想备份？** 整个状态都在 `data/market_scan.db` 一个文件里（含持仓），
  连同 `config.yaml` 一起备份即可。
- **银行股为什么总是"质量未达标"？** 银行没有"毛利率"概念（yfinance 无此字段），
  会在毛利率一项失败——这与巴菲特"看不懂的不买"原则一致；若你想纳入银行股，
  可单独人工分析，或在 config 里降低标准后用 `--tickers BAC,JPM` 单独扫。
- **录错交易了？** `python main.py trades` 找到编号，`python main.py delete-trade <id>` 删除。
